为外部过程配置连接描述符

为外部过程配置连接描述符

外部过程是从另一个程序调用的、用不同语言编写的过程。PL/SQL 程序为执行特殊处理而调用一个或多个所需的 C 语言例行程序，便是这样的示例。

为了使 Oracle 数据库能够连接到外部过程，必须用网络服务名配置数据库服务器以访问外部过程。

注：Oracle Net Configuration Assistant 在安装过程中将自动创建用于外部过程的网络服务名。

要配置外部过程的数据库服务器，请执行以下操作：

  1. 按照为监听程序配置外部过程中的说明，确保为外部过程配置了监听程序。

  2. 在数据库的 tnsnames.ora 文件中创建一个网络服务名。请确保连接描述符信息与为此监听程序配置的信息相匹配。

  3. 选择文件 > 保存网络配置。

为客户机配置默认域

为客户机配置默认域

在客户机通常从特定域请求名称的环境中，十分适合设置默认域。如果在概要文件中设置了默认域，则默认域会自动附加到任何未限定的网络服务名或服务名。例如，如果默认域设置为 us.example.com，则以下连接字符串中的 sales 将作为 sales.us.example.com 进行搜索：

CONNECT scott/tiger@sales

要指定默认域，请执行以下操作：

  1. 在导航器窗格中，展开 Oracle 网络配置 > 本地 > 概要文件。

  2. 在右窗格的列表中，选择命名。

  3. 选择方法选项卡。

  4. 请在缺省域中输入该域。

  5. 选择文件 > 保存网络配置。

配置高级概要文件信息

配置高级概要文件信息

概要文件可以配置为具有下列高级功能：

  • 连接超时值

  • 日志文件中唯一的客户机标识符

  2. 从右侧窗格的列表中选择一般信息。

  3. 选择高级选项卡。

  4. 输入要设置的选项值。

  5. 选择文件 > 保存网络配置。

配置加密

配置加密

加密是将消息伪装使其不能被指定收件人以外的任何人读取的过程。

要配置加密参数，请执行以下操作：

  1. 在导航器窗格中，展开 Oracle 网络配置 > 本地 > 概要文件。

  2. 从右侧窗格的列表中，选择 Oracle Advanced Security。

  3. 选择加密选项卡。

  4. 从加密列表中选择客户机或服务器。

  5. 从加密类型列表中，选择一个值以指定协商加密和完整性时的客户机或服务器行为。下面是四个可能的值：

    元素

    说明
    被接受

    如果连接的对方指定了“必需的”或“请求的”，并且对方存在可用的一致算法，则服务将是活动的；否则服务将是不活动的。

    被拒绝

    		 服务必须不是活动的，如果另一方指定“必需的”，则连接将会失败。
    请求的

    如果连接的对方指定了“被接受”、“必需的”或“请求的”，并且对方存在可用的一致算法，则服务将是活动的；否则服务将是不活动的。
    必需的 服务必须是活动的，如果另一方指定“拒绝”或没有一致的算法，则连接将会失败。
  6. 在“加密种子”字段中输入加密种子：最多为 256 个字符的随机字符串。此加密种子与下一步中选择的加密算法一起使用。
  7. 从“可用方法”列表中选择验证算法，然后选择右箭头 (<) 按钮将其移动到“所选方法”列表。“所选方法”列表中方法的顺序决定了协商的首选顺序。Oracle Advanced Security 支持以下算法：

    元素

    说明
    RC4_256

    256 位密钥大小

    3DES168

    		 具有三密钥 (168 位) 选项的“三重 DES”
    RC4_128

    128 位密钥大小
    3DES112 具有双密钥 (112 位) 选项的“三重 DES”
    RC4_56 56 位密钥大小
    DES 标准 56 位密钥大小
    RC4_40 40 位密钥大小
    DES40 40 位密钥大小
  8. 选择文件 > 保存网络配置。

注：所有 Oracle Advanced Security 加密算法和密钥长度仅适用于美国本土。

配置数据完整性

配置数据完整性

数据完整性校验是根据消息内容计算数量的过程。消息的发送方计算校验和。校验和的值将作为消息包的一部分。消息的接收方重新计算所收到消息的校验和。如果传输过程中消息未改变，则这两个校验和的值必定完全一样。

要配置数据完整性，请执行以下操作：

  1. 在导航器窗格中，展开 Oracle 网络配置 > 本地 > 概要文件。

  2. 从右侧窗格的列表中，选择 Oracle Advanced Security。

  3. 选择完整性选项卡。

  4. 通过分别选择“客户机”或“服务器”复选框启用客户机或服务器的数据完整性。

  5. 指定客户机和服务器将如何协商数据的完整性。必须选择客户机和服务器的校验和级别。级别说明如下：

    元素

    说明
    被接受

    如果连接的对方指定了“必需的”或“请求的”，并且对方存在可用的一致算法，则服务将是活动的；否则服务将是不活动的。

    被拒绝

    		 服务必须不是活动的，如果另一方指定“必需的”，则连接将会失败。

    请求的

    如果连接的对方指定了“被接受”、“必需的”或“请求的”，并且对方存在可用的一致算法，则服务将是活动的；否则服务将是不活动的。

    必需的

    		 服务必须是活动的，如果另一方指定“拒绝”或没有一致的算法，则连接将会失败。
  6. 选择文件 > 保存网络配置。

配置监听程序地址

配置监听程序地址

监听程序必须配置有一个或多个协议地址。监听程序从配置有相同地址的客户机那里接收连接请求。

要配置监听程序的协议地址，请执行以下操作：

  1. a. 在导航器窗格中，展开 Oracle 网络配置 > 本地 > 监听程序。

  2. 选择监听程序。

  3. 从右边窗格的列表中选择监听程序位置。

  4. 选择添加地址。此时将显示新的地址选项卡。

  5. 选择一个协议，然后为所选协议输入相应的协议参数信息。

    当配置监听程序对 Oracle Net 客户机的 TCP/IP 进行监听时，应该选择默认端口或 1521 端口。如果不这样做，则必须在初始化参数文件中配置 LOCAL_LISTENER 参数并通过命名方法解析监听程序名。

  6. 如果计算机有多个 IP 地址，而且希望监听程序监听所有可用的 IP 地址，请配置 TCP/IP 或使用 SSL 的 TCP/IP，并在主机字段中输入该计算机的主机名。

  7. 还可以单击显示高级指定相应的设置，以此指定发送和接收会话操作的 I/O 缓冲区空间。

  8. 选择文件 > 保存网络配置。

有关配置非默认端口的详细信息，请参阅 Oracle10i Net Administrator's Guide 中的第 10 章“Configuring and Administering the Listener”。

有关配置 I/O 缓冲区空间的详细信息，请参阅 Oracle10i Net Administrator's Guide 中的第 14 章“Optimizing Performance”。

配置监听程序的事件记录和跟踪

配置监听程序的事件记录和跟踪

可以使用事件记录和跟踪参数配置监听程序。

日志信息通常存储在 listener.log 文件中。在 UNIX 上，该文件在 $ORACLE_HOME/network/log 目录中；在 Window 平台上，该文件在 ORACLE_HOME\network\log 目录中。

跟踪信息默认存储在 listener.trc 文件中。在 UNIX 上，该文件在 $ORACLE_HOME/network/trace 目录中；在 Window 平台上，该文件在 ORACLE_HOME\network\trace 目录中。

要启用事件记录或跟踪，请执行以下操作：

  1. a. 在导航器窗格中，展开 Oracle 网络配置 > 本地 > 监听程序。

  2. 选择监听程序。

  3. 在右侧窗格的列表中，选择常规参数。

  4. 选择事件记录和跟踪选项卡。

  5. 根据需要调整事件记录和跟踪设置。

  6. 选择文件 > 保存网络配置。

配置监听程序的口令验证

配置监听程序的口令验证

可以配置监听程序的安全性。这样，从监听程序控制实用程序使用的权限操作 (例如，保存配置更改或停止监听程序) 将需要口令。Oracle Net Manager 使您能够设置加密口令。

要为监听程序设置口令，请执行以下操作：

  1. a. 在导航器窗格中，展开 Oracle 网络配置 > 本地 > 监听程序。

  2. 选择监听程序。

  3. 在右侧窗格的列表中，选择常规参数。

  4. 选择验证选项卡。

  5. 选择需要监听程序操作口令。

  6. 在口令字段中输入口令，并在确认口令字段中再次输入此口令。

  7. 选择文件 > 保存网络配置。

配置概要文件事件记录和跟踪

配置概要文件事件记录和跟踪

可配置客户机以进行事件记录和跟踪。

日志信息通常存储在 sqlnet.log 文件中。在 UNIX 上，该文件在 $ORACLE_HOME/network/log 目录中；在 Window 平台上，该文件在 ORACLE_HOME \network\log 目录中。

默认情况下，客户机的跟踪信息存储在 sqlnet.trc 中，服务器的跟踪信息存储在 svr_pid..trc 中。在 UNIX 上，这两个文件位于 $ORACLE_HOME/network/trace 目录中；在 Windows 平台上，这两个文件位于 ORACLE_HOME \network\trace 目录中。

要启用事件记录或跟踪，请执行以下操作：

  1. 在导航器窗格中，展开 Oracle 网络配置 > 本地 > 概要文件。

  2. 从右侧窗格的列表中选择一般信息。

  3. 选择事件记录或跟踪选项卡。

  4. 指定“事件记录”或“跟踪”设置

  5. 选择文件 > 保存网络配置。
