关于 Oracle Advanced Security

Oracle Advanced Security 为 Oracle 环境添加了一组综合安全功能。此组安全功能保护企业网络并将公司网络安全扩展到 Internet。它提供将网络加密及验证解决方案、一次登入服务和安全协议集成的单一来源。Oracle Advanced Security 集成了工业标准并使 Oracle 网络及其以外产品具有无可比拟的安全性。

已接受

我方的连接不需要安全服务，但如果对方要求一组必需的或请求的安全服务，则允许我方启用安全服务。如果对方设置为必需的或请求的，而且找到了算法匹配，将继续进行正确的连接，并启用安全服务。如果对方设置为必需的，却未找到算法匹配，连接将终止，并出现错误消息 ORA- 2650。

ADDRESS

ADDRESS

网络对象的网络协议地址。

建立连接后，客户机和请求的接收方 (监听程序、Oracle Names Server 或 Oracle Connection Manager) 将配置有相同的协议地址。客户机使用该地址向特定的网络对象位置发送连接请求，同时接受方在该地址“监听”请求。

下列地址定义监听程序的监听地址。

ADDRESS=

(PROTOCOL=tcp)

(HOST=sales-server)

(PORT=1521))

地址中的协议参数取决于所选的协议。

“高级服务选项”对话框

“高级服务选项”对话框使您能够配置高级 CONNECT_DATA 参数。

其他服务设置

元素

说明
实例名

输入要访问的数据库实例的名称。实例名可以从初始化参数文件的 INSTANCE_NAME 参数值中获取。
会话数据单元 (SDU) 输入会话数据单元大小以更改与跨网络发送的数据包有关的性能特征。将 SDU 大小设置为正常传输的帧大小的倍数。正常的以太网帧大小为 1500，不过可用的部分为 1460。如果不使用加密，则保留一个字节，因此使用的算法为 1460 的倍数加 1；如果对以太网进行了加密，则使用的算法为 1460 的倍数。
用于异构服务 选择此项以便 Oracle 服务器通过异构服务访问非 Oracle 系统。

更改 Oracle Context

更改 Oracle Context

启动 Oracle Net Manager 时，目录 > 服务命名文件夹中显示的对象是包含在默认的 Oracle Context 中的 Oracle 服务条目，它们是在目录用法配置期间使用 Oracle Net Configuration Assistant 建立的。您可以创建和修改另一个您已验证的 Oracle Context 的条目。

要更改当前的 Oracle Context，请执行以下操作：

  1. 选择命令 > 目录 > 更改当前上下文。将显示选择新上下文对话框。

  2. 从目录命名上下文列表中，选择包含要查看的 Oracle Context 的目录条目。

  3. 从 Oracle Context 列表中，选择 Oracle Context，然后选择确定。如果修改没有保存到当前 Oracle Context 中，则“保存”对话框将提示您保存更改。

客户机负载平衡

客户机负载平衡

如果多个监听程序为单个数据库服务，那么客户机可为其连接请求随机选择监听程序，这称为负载平衡。此随机选择使所有监听程序以共同分担为传入连接请求提供服务所带来的负担。
