创建网络服务别名

创建网络服务别名

注释：

  • 只有 OracleNetAdmins 或 OracleContextAdmins 组的成员用户可以在目录中创建网络服务别名条目。

  • 网络服务别名在使用 Microsoft Active Directory 时不受支持。

  • 可以从 Oracle Names Server 中导出现有的别名对象。有关详细信息，请参阅 Oracle10i Net Services Administrator's Guide 中的第 10 章“Exporting Naming Data to a Directory Server”。

可以在目录服务器中，为目录命名对象 - 网络服务名或数据库服务 - 创建网络服务别名。网络服务别名用于存储它所引用的对象的名称。

当客户机请求网络服务别名的目录查找时，此目录确定此条目为网络服务别名，然后象查找所引用的条目那样完成查找。例如，可以为网络服务名 sales 创建网络服务别名 salesalias。如果按 CONNECT scott/tiger@salesalias 方式使用 salesalias 连接数据库服务，它将被真正进行解析并使用 sales 的连接描述符信息。

网络服务别名有两种主要用途：

  • 使用网络服务名别名，以便客户机可按另一名称引用数据库服务或网络服务名。

  • 在 Oracle Context 中使用数据库服务的网络服务名别名或在其他的 Oracle Context 中使用网络服务名。这样，可以在其他的 Oracle Context 中使用目录服务器中定义数据库服务或网络服务名。

要创建网络服务别名，请执行以下操作：

  1. 在导航器窗格中，展开 Oracle 网络配置 > 目录 > 服务命名。

  2. 从工具栏中选择加号 (+)，或从菜单栏中选择编辑 > 创建。将显示创建网络服务别名对话框。

  3. 在 网络服务别名字段中，输入网络服务别名的名称。

  4. 在“网络服务或数据库服务”框中，输入以下有关网络服务名或数据库服务的信息：
    • Oracle Context：从列表中选择一个值或在字段中输入一个值，以此指定数据库服务或网络服务名的 Oracle Context。
    • 名称：从列表中选择一个值或在字段中输入一个值，以此指定数据库服务或网络服务名的唯一判别名 (DN)。
  5. 选择创建。

SSL 概要文件

使用 SSL 选项卡可修改“安全套接字层 (SSL)”设置。SSL 是一个保护网络通信的工业标准协议。SSL 提供验证、加密和数据完整性。使用 SSL 可保护任意客户机和服务器之间的通信。具有特别意义的是，可以使用 SSL 进行任意客户机或服务器到一个或多个 Oracle 服务器的验证，或者是一个 Oracle 服务器到任意客户机的验证。

撤销检查 (仅限服务器)

为证书指定撤销检查。选择下列某个值：

  • 无：选择该值可禁用证书的撤消检查。

  • 必需：选择此项可撤销可用的证书。如果撤销证书后，没有找到相应的“证书撤销列表 (CRL)”，则拒绝 SSL 连接。如果找不到相应的 CRL 来确定证书的撤销状态，而且证书尚未被撤销，则接受 SSL 连接。

  • 请求：选择该值可在找到 CRL 的情况下撤消证书。如果证书已撤消，则拒绝 SSL 连接。如果找不到相应的 CRL 来确定证书的撤消状态，而且证书尚未被撤消，则接受 SSL 连接。

  • 是：选择该值可检查服务器 DN。如果 DN 与服务名匹配，则连接成功。如果 DN 与服务名不匹配，仍可成功连接，但在 sqlnet.log 文件中记录一条错误。

  • 否：选择该值则不检查服务器 DN。忽略此检查，可使服务器掩饰自己的身份。

“地址列表选项”对话框

“地址列表选项”对话框

使用“地址列表选项”对话框可以指定顺序，并指定连接描述符中的协议地址在连接中是如何使用的。

按顺序尝试每个地址, 直到成功

选择以启用 Oracle9i 或 Oracle8i 客户机的连接时故障转移。如果不能连接到第一个地址，故障转移会指示客户机使用下一个监听程序协议地址。选择此选项以及使用与 Net8 8.0 客户机兼容的选项选项，面向 8.0 发行版或以前的客户机关闭源路由选择。

随机尝试每个地址，直到成功

选择以启用 Oracle9i 或 Oracle8i 客户机的客户机负载平衡和连接时故障转移。

尝试每个地址，随机选择

选择以启用 Oracle9i 或 Oracle8i 客户机的客户机负载平衡功能。

顺序使用每个地址直到到达目标为止。

选择以启用 Oracle8i 或 Oracle8 客户机的源路由选择。

当达到目标需要多个中继地址时请选择该选项。该功能通常用于启用 Oracle Connection Manager 功能。

Oracle9i 客户机不需要此参数。

只使用第一个地址

选择此选项以禁用源路由选择、连接时故障转移和客户机负载平衡功能。

使用与 Net8 8.0 客户机兼容的选项

如果正在配置发行版 8.0 客户机，请选择此选项。您只能从下列选项中选择：

- 顺序尝试每个地址，直到成功

顺序使用每个地址直到到达目标为止。

如果正在配置 Oracle9i 或 Oracle8i 客户机，请清除此选项。

为监听程序配置外部过程

为监听程序配置外部过程

外部过程是从另一个程序调用的、用不同语言编写的过程。PL/SQL 程序为执行特殊处理而调用一个或多个所需的 C 语言例行程序，便是这样的示例。

可以配置监听程序来监听外部过程的调用情况。当应用程序调用外部过程时，监听程序启动默认情况下名为 extproc 的外部过程代理。应用程序使用由监听程序建立的网络连接，向该代理传递以下信息：

  • DLL 或共享库名称

  • 外部过程名

  • 任意参数

接下来，该代理将加载 DLL 和运行外部过程，并将外部过程返回的任何值传回应用程序。

代理可以驻留在数据库服务器所在的同一计算机上，也可以驻留在监听程序所在的远程计算机上。

对于连接到外部过程的 Oracle 数据库而言，请使用协议地址和外部过程服务信息配置 listener.ora 文件。

注：在安装 Oracle9i 或 Oracle8i 期间，Oracle Net Configuration Assistant 将在 listener.ora 文件中配置此信息。

要配置监听程序来监听外部过程的调用情况，请执行以下操作：

  1. 创建一个专门用于处理外部过程的监听程序：

  2. a. 在导航器窗格中，展开 Oracle 网络配置 > 本地 > 监听程序。

    b. 在工具栏中选择加号 (+) 或选择编辑 > 创建。此时将显示“选择监听程序名”对话框。

    c. 在监听程序名字段中输入唯一的监听程序名。

    d. 选择确定。

  3. 如果代理驻留在数据库服务器上，请在 listener.ora 文件中配置 TCP/IP 或 IPC 协议地址。如果外部过程代理驻留在远程计算机上，请在 listener.ora 文件中配置 TCP/IP 协议地址。以下过程描述了如何为默认外部过程代理 extproc 创建 IPC 地址：

  4. a. 选择刚创建的监听程序。

    b. 从右侧窗格的列表中选择监听位置。

    c. 选择添加地址。此时将显示新的地址选项卡。

    d. 从协议列表中选择 IPC，然后为密钥输入值。

  5. 在 listener.ora 文件中添加有关 extproc 的服务信息

  6. a. 从右侧窗格的列表中，选择其他服务。

    b. 选择添加服务。此时将显示新的服务选项卡。

    c. 在程序名字段中输入 extproc，在此字段中输入例如 extproc 这样的系统标识符，并在 Oracle 主目录字段中输入 extproc 可执行文件所在的 Oracle 主目录。如果应用程序要求在启动代理之前先设置环境变量，则在环境字段中输入值。例如，如果应用程序要求将环境变量 MYAPPDIR 用于二进制位置，并将 APPID 用于 ID，则可以在环境字段中输入以下值：

    'MYAPPDIR=/myappdir/bin','APPID=MYAPP'

  7. 如果配置了 TCP/IP 协议地址，请指定允许访问的远程客户机：

  8. a. 在导航器窗格中，展开本地 > 概要文件。

    b. 从右侧窗格的列表中，选择一般信息。

    c. 选择访问权限选项卡。

    d. 选择检查 TCP/IP 客户机访问权限选项。

    e. 在允许访问的客户机字段中，输入希望包含或排除的客户机的主机名或 IP 地址，放置在同一行上的条目之间用逗号分隔。

  9. 在数据库服务器的 tnsnames.ora 文件中创建网络服务名，确保其连接描述符与 listener.ora 文件中配置的信息相匹配，如为外部过程配置连接描述符中所述。

  10. 选择文件 > 保存网络配置。

配置高级连接数据选项

除了数据库的服务名或 Oracle 系统标识符 (SID) 以外，还可以选择使用以下选项配置连接描述符的 CONNECT_DATA 部分：

  • 连接特定数据库实例的实例名

  • 数据库的 Oracle 主目录，用于通过网络启动数据库

  • 指定缓冲区大小的会话数据单元大小

  • 连接专用服务器进程的专用服务器

  • 访问非 Oracle 系统的异构服务

  • 访问 Oracle Rdb 数据库的 Oracle Rdb 设置

要配置网络服务名的高级连接选项，请执行以下操作：

  1. 在导航器窗格中，展开 Oracle 网络配置 > 目录 > 本地命名/服务命名。

  2. 选择网络服务名或数据库服务。右侧窗格显示当前目标服务和地址列表。

  3. 在服务标识框中，选择高级。将显示高级服务选项对话框。

  4. 输入适当的字段或选择适当的选项，然后选择确定。

  5. 在右侧窗格中，选择应用。

  6. 如果要对本地文件夹进行上述更改，请选择文件 > 保存网络配置。
