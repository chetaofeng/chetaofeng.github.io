链接限定词

链接限定词

附加在全局数据库链接后面，为数据库用户名和口令身份证明提供替代设置的限定词。例如，可以将链接限定词 emp 附加在全局数据库链接 sales.us.example.com 的后面。到具有全局数据库链接和链接限定词的到远程数据库的连接将是：

CONNECT scott/tiger@sales.example.com@emp

监听程序

监听程序

驻留在服务器上的进程，用于监听传入的客户机连接请求和管理传输到服务器的通信量。

每次客户机请求与服务器建立网络会话时，监听程序接收实际的请求。如果客户机信息与监听程序信息匹配，则监听程序将允许其连接到服务器。

监听程序配置概览

安装期间的配置

在数据库服务器上安装企业版或标准版期间，Oracle Net Configuration Assistant 将配置名为 LISTENER 的监听程序，该程序中含有 Oracle 数据库的 TCP/IP 监听协议地址。在进行定制安装期间，Oracle Net Configuration Assistant 将提示您为自己选取的监听程序配置监听程序名和协议地址。

另外，无论采用何种安装类型，外部过程调用的监听 IPC 协议地址都是自动配置的。

Oracle Net Configuration Assistant 还自动在 listener.ora 文件中配置外部过程的服务信息。

具有 Oracle Net Manager 的配置

Oracle Net Manager 使您能够完成以下操作：

  • 创建监听程序

  • 配置 Oracle8 或以前数据库的服务信息

  • 配置非 Oracle 数据库服务的服务信息，包括外部过程和异构服务

  • 启用跟踪和事件记录功能

  • 为监听程序控制实用程序启用口令验证

要访问监听程序配置，请执行以下操作：

在导航器窗格中，展开 Oracle Net 配置 > 本地 > 监听程序。

有关详细信息，请参阅 Oracle10i Net Administrator's Guide 中的第 10 章“Configuring and Administering the Listener”。

监听程序概览

监听程序概览

在服务器 (它代表客户机应用程序接收连接请求) 上的监听程序、进程的帮助下，服务可以协调其会话。配置监听程序，使其在数据库服务或非数据库服务的协议地址上进行“监听”。

协议地址是在数据库服务或非数据库服务的监听程序配置文件 listener.ora 中配置的。Oracle9i 或 Oracle8i 数据库服务将其信息自动注册到监听程序，如服务名、实例名及加载信息。这种名为服务注册的功能不需要在 listener.ora 文件中进行服务配置。其他服务 (包括 Oracle8 和 Oracle7 数据库服务以及 Oracle Enterprise Manager 管理工具) 需要在 listener.ora 文件中配置服务。

一旦配置了监听程序，就可以用监听程序控制实用程序管理它。

本地命名概览

本地命名概览

本地命名是主要命名方法中的一种，它用于将简称 (网络服务名) 解析为连接描述符，客户机使用此连接描述符连接数据库或服务。本地命名在本地 tnsnames.ora 文件中存储网络服务名及其连接描述符。

最终用户以下列格式输入包括网络服务名的连接字符串：

CONNECT username/password@net_service_name

在下面的示例中，sales 是在 tnsnames.ora 文件中找到的网络服务名：

CONNECT scott/tiger@sales

返回到客户机的连接描述符包括：

  • 监听程序协议地址的位置

  • 数据库的服务名

客户机使用此信息连接到某个监听程序。用同一个地址信息配置的监听程序接收此请求。如果监听程序正在监听对同一数据库服务的请求 (该数据库服务在连接描述符中指定)，则授予连接权限。

有关详细信息，请参阅 Oracle10i Net Administrator's Guide 中的第 2 章“Connectivity Concepts”。

listener.ora 文件

listener.ora 文件

指定以下各项的监听程序的配置文件：

  • 监听程序名

  • 它正在接受连接请求的协议地址

  • 监听程序正在监听的服务 (数据库或非数据库服务)

listener.ora 文件必须驻留在以下某一位置：

  1. 由环境变量 TNS_ADMIN 指定的目录。如果在 Windows NT 上没有将 TNS_ADMIN 环境变量定义为变量，它可能在注册表中。

  2. 节点的全局配置目录。对于 Sun Solaris，此目录为 /var/opt/oracle。Windows NT 不具有中央目录。

  3. 在 UNIX 上为 $ORACLE_HOME/network/admin，而在 Windows 操作系统上为 ORACLE_HOME\network\admin 目录。

注：Oracle9i 和 Oracle8i 由于服务注册而不需要标识数据库服务。不过，如果您打算使用 Oracle Enterprise Manager 来监视数据库，则需要静态服务配置。

本地命名配置概览

“管理员”安装或“运行时”安装

在“管理员”或“运行时”客户机安装过程中，如果尚未配置目录用法，Oracle Net Configuration Assistant 将提示您在 tnsnames.ora 文件中配置网络服务名。

定制安装

如果在客户机或服务器定制安装期间选择本地命名作为命名方法，Oracle Net Configuration Assistant 将提示您在 tnsnames.ora 文件中配置网络服务名。

具有 Oracle Net Manager 的配置

Oracle Net Manager 使您能够配置网络服务名以及：

  • 创建非 Oracle 数据库服务的网络服务名

  • 通过多个协议地址配置到监听程序的多个路由

  • 配置数据库服务或网络服务名的高级 CONNECT_DATA 信息。

要访问本地命名配置，请执行以下操作：

从导航器窗格中选择本地 > 服务命名文件夹，以便创建网络服务名或修改连接描述符信息。

有关详细信息，请参阅 Oracle10i Net Administrator's Guide 中的第 8 章“Configuring Naming Methods”。
