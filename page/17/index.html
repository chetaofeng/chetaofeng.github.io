一般参数：事件记录和跟踪

使用事件记录和跟踪选项卡可以启用监听程序的事件记录和跟踪。Oracle Net Manager 使您能够指定写入监听程序日志文件的名称和位置。

禁用事件记录

选择此选项以禁用监听程序上的事件记录。

启用事件记录

选择此选项以启用监听程序上的事件记录。一旦启用，您就可以指定写入日志信息的位置。

日志文件

输入将写入监听程序日志信息的目录路径名和文件名。请注意该文件的扩展名总是 .log。默认名称是 listener.log。在 UNIX 上，默认目录是 $ORACLE_HOME/network/log；在 Windows 平台上，默认目录是 ORACLE_HOME\network\log。

禁用跟踪

选择此选项以禁用监听程序上的跟踪。

启用跟踪

选择此选项以启用监听程序上的跟踪。一旦启用跟踪，您就可以指定需要的跟踪级别，并通过更改默认跟踪文件来指定写入跟踪信息的位置。

跟踪级别

从列表中，选择跟踪级别：

关闭 - 尚未启用跟踪。OFF 是默认设置。

用户 - 将跟踪设置为与用户相应的级别。跟踪以标识用户导致的错误条件。

管理员 - 将跟踪设置为与数据库管理员相应的级别。跟踪以标识特定的安装问题。

支持 - 将跟踪设置为与客户支持相应的级别。

跟踪文件可能会变得很大。Oracle Corporation 建议在不进行网络问题诊断时关闭跟踪功能。

跟踪文件

输入监听程序跟踪信息将写入的目录路径名和文件名。请注意该文件的扩展名总是 .trc。默认名称为 listener.trc。在 UNIX 上，默认目录为 $ORACLE_HOME/network/trace。在 Windows 平台上，默认目录为 ORACLE_HOME\network\trace。

一般信息：跟踪

跟踪选项卡使您能够设置本计算机上的跟踪参数。如果本计算机是客户机，请使用客户机信息字段。如果本计算机是服务器，请使用服务器信息字段。

客户机跟踪保持对客户机执行的操作进行跟踪。服务器跟踪保持网络连接过程中的服务器执行的跟踪操作。

跟踪级别

从列表中，选择跟踪级别：

OFF - 尚未启用跟踪。OFF 是默认设置。

USER - 将跟踪设置为与用户相应的级别。跟踪以标识用户导致的错误条件。

ADMIN - 将跟踪设置为与数据库管理员相应的级别。跟踪以标识特定的安装问题。

SUPPORT - 将跟踪设置为与客户支持相应的级别。

跟踪文件可能会变得很大。Oracle Corporation 建议在不进行网络问题诊断时关闭跟踪功能。

跟踪目录

输入跟踪文件将写入的目录。当客户机和服务器跟踪文件传送有关客户机或服务器的信息时，这两种信息都将写入本地计算机中。默认值为 $ORACLE_ HOME/network/trace (在 UNIX 上) 和 ORACLE_HOME\network\trace (在 Windows 平台上)。

跟踪文件

输入跟踪文件名。默认情况下，客户机跟踪文件名为 sqlnet.trc，服务器跟踪文件名为 svr_pid..trc。

唯一跟踪文件名称

选择此选项以创建每个跟踪文件的唯一名称。在每个跟踪文件名后追加进程标识符以便多个跟踪文件可以共存。如果您不想创建唯一的跟踪文件，并希望所有的跟踪信息都存储在相同的跟踪文件中，请清除此选项。

异构服务

异构服务

一种集成组件，它提供用于从 Oracle 数据库服务器访问非 Oracle 系统的一般技术。异构服务使您能够进行以下操作：

  • 使用 Oracle SQL 可透明地访问存储在非 Oracle 系统中的数据，就好象数据驻留在 Oracle 服务器中一样。

  • 使用 Oracle 过程调用从 Oracle 分布式环境透明地访问非 Oracle 系统、服务或应用程序编程接口 (API)。

要与非 Oracle 系统建立连接，Oracle8i 数据库会通过网关上的监听程序启动一个代理进程。为使 Oracle8i 数据库连接到这个代理，服务器必须配置有网络服务名以访问代理，而且监听程序必须配置有该代理的服务信息。

监听程序控制实用程序

监听程序控制实用程序

包含在 Oracle Net Services 中的实用程序，用来控制各种监听程序功能，如启动、停止以及获取监听程序的状态。监听程序控制实用程序以下列方式执行：

lsnrctl command [listener_name]

[listener_name] 是在 listener.ora 文件中定义的监听程序名。如果正在使用默认的监听程序名 LISTENER，则不必标识监听程序。

有关详细信息，请参阅 Oracle10i Net Services Reference Guide 中的第 1 章“Listener Control Utility”。

监听位置

监听位置

配置监听程序以在特定的网络协议地址上监听某个数据库的传入连接请求。监听位置选项卡使您能够配置一个或多个监听位置地址，监听程序将在这些地址上等待客户机的连接。

协议

从列表中,选择地址协议。根据所选协议的不同，在域中输入相应的协议参数信息。

“显示高级/隐藏高级”按钮

“显示高级/隐藏高级”按钮可进行切换，以显示或隐藏为监听程序指定的 I/O 缓冲区空间限制的值。

TCP/IP、启用 SSL 的 TCP/IP 以及 SDP 协议都支持这些设置。

  • 发送缓冲区大小总计

以字节为单位，指定会话的发送操作的缓冲区空间。

因为数据库服务器向客户机写入数据，所以一般在服务器端进行设置比较合适。如果数据库服务器要接收大量请求，则还要设置接收缓冲区大小总计的值。

  • 接收缓冲区大小总计

以字节为单位，指定会话的接收操作的缓冲区空间。

使该地址专用于 JServer 连接

选择使用用于 Oracle JServer 发行版 8.1 连接的通用 ORB 间协议 (GIOP) 端点来配置 listener.ora 文件。

添加地址

选择此选项以创建新的协议地址。将显示一个新的地址选项卡，在选项卡中可以选择协议，并在域中输入协议信息。

删除地址

选择此选项以删除所选的地址选项卡及其地址信息。

有关配置 I/O 缓冲区空间的详细信息，请参阅 Oracle10i Net Administrator's Guide 中的第 14 章“Optimizing Performance”。
