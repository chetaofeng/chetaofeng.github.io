“选择新上下文”对话框

“选择新上下文”对话框

“选择新上下文”对话框使您能够设置另一个 Oracle Context，后者是目录信息树 (DIT) 中名为 cn=OracleContext 的容器，其中包含用于 Oracle 软件的条目。这些条目可以包含网络服务名和连接到数据库服务的数据库服务名。默认情况下，Oracle Net Manager 使用由 Oracle Net Configuration Assistant 配置的 Oracle Context。

目录命名上下文

从列表中，选择包含您要查看的 Oracle Context 的目录条目。

Oracle Context

从此列表中选择一个 Oracle Context。您选择的 Oracle Context 应包含目录树中您打算修改或添加其网络信息的条目。

“导航器”窗格中的目录 > 服务命名文件夹随所选的 Oracle Context 的网络对象进行更新。

为监听程序配置异构服务

为监听程序配置异构服务

异构服务使您能够从 Oracle 服务器访问非 Oracle 系统。要启动与非 Oracle 系统的连接，Oracle 服务器需要通过网关上的监听程序启动代理进程。监听程序的“异构服务配置”包括创建“异构”代理的服务信息。

要为监听程序配置异构服务，请执行以下操作：

  1. a. 在导航器窗格中，展开 Oracle 网络配置 > 本地 > 监听程序。

  2. 选择监听程序。

  3. 从右侧窗格的列表中，选择其他服务。

  4. 选择添加服务。此时将显示新的服务选项卡。

  5. 请在程序名字段中输入代理可执行文件，在 SID 字段中输入代理的 Oracle 系统标识符 (SID)，并在 Oracle 主目录字段中输入 Oracle 主目录。

  6. 确保服务器配置有连接描述符以访问异构服务，如为外部过程配置连接描述符中所述。

  7. 选择文件 > 保存网络配置。

配置 Oracle Advanced Security

配置 Oracle Advanced Security

使用 Oracle Net Manager 配置 Oracle Advanced Security。每个 Oracle Advanced Security 选项卡页都使您能够配置一组单独的参数。Oracle Advanced Security 将启用数据加密和完整性检查、增强验证以及一次登入。Oracle Advanced Security 还在符合 LDAP 的目录服务器上提供集中式用户管理，并提供基于证书的一次登入功能，该功能依赖于“安全套接字层 (SSL)”。

要配置 Oracle Advanced Security，请执行以下操作：

  1. 在导航器窗格中，展开 Oracle 网络配置 > 本地 > 概要文件。

  2. 从右侧窗格的列表中，选择 Oracle Advanced Security。

  3. 将显示下列选项卡面板：

    • 验证

    • 其他参数

    • 完整性

    • 加密

    • SSL

  5. 更改您的配置。

  6. 选择文件 > 保存网络配置。

更改将保存到 sqlnet.ora 文件中。

注：如果列表中没有显示 Oracle Advanced Security，则说明它尚未安装。

有关详细信息，请参阅 Oracle Advanced Security Administrator's Guide。

配置连接到 Oracle Connection Manager 的连接描述符

配置连接到 Oracle Connection Manager 的连接描述符

要通过 Oracle Connection Manager 将客户机路由至数据库服务器，请配置 tnsnames.ora 文件或带有指定 Oracle Connection Manager 的协议地址的连接描述符的目录服务器。客户机可以使用此地址连接到 Oracle Connection Manager 所在的计算机。

要配置 Oracle Connection Manager 的协议地址，请执行以下操作：

  1. 在导航器窗格中，展开 Oracle 网络配置 > 目录/本地 > 服务命名。

  2. 从工具栏中选择加号 (+)，或从菜单栏中选择编辑 > 创建。将显示“网络服务名向导”的“欢迎使用”页。

  3. 在网络服务名字段中输入任意名称。

  4. 选择下一步。此时将显示“协议”页。

  5. 选择 Oracle Connection Manager 被配置在其上监听的协议。默认情况下，该协议为 TCP/IP。

  6. 选择下一步。此时将显示“协议设置”页。

  7. 在提供的字段中为所选协议输入相应的协议参数。如果正在使用 TCP/IP，则默认端口是 1630。

  8. 选择下一步。此时将显示“服务”页。

  9. 如果目标服务是 Oracle8i 或更高版本的服务，请选择 Oracle8i 或更高版本，然后在服务名字段中输入服务名。如果目标服务是 Oracle8 或 Oracle7，请选择 Oracle8 或更低版本，然后在数据库 SID 字段中输入 Oracle 系统标识符。

  10. 选择下一步。此时将显示“服务”页。

  11. 单击完成保存配置并关闭“网络服务名向导”。

  12. 选择文件 > 保存网络配置。

  13. 新的网络服务名将添加到“服务命名”文件夹。

有关全面启用 Oracle Connection Manager 功能的详细信息，请参阅 Oracle10i Net Administrator's Guide 中的第 11 章“Configuring and Administering Oracle Connection Manager”。

配置多个地址选项

配置多个地址选项

当数据库服务可被多个监听程序的协议地址访问时，指定协议地址的使用顺序是非常重要的。例如，可以随机选择协议地址，或者按顺序尝试。

当为网络服务名或数据库服务配置了多个协议地址时，则以下选项是可配置的：

  • 连接时故障转移用来选择第一个协议地址，并在第一个协议地址失败时切换到下一个协议地址。

  • 客户机负载平衡用来随机选择协议地址。

  • 源路由选择用来按顺序使用所有协议地址。

要配置多个地址选项，请执行以下操作：

  1. 在导航器窗格中，展开 Oracle 网络配置 > 目录 > 本地或服务命名。

  2. 选择网络服务名或数据库服务。右侧窗格显示当前目标服务和地址列表。

  3. 在地址配置框中，选择高级。此时将显示地址列表选项对话框。

  4. 如果此配置适用于 Oracle9i 或 Oracle8i 客户机，请清除使用与 Net8 8.0 客户机兼容的选项。

  5. 选择协议地址列表的相应选项，然后选择确定。

  6. 在右侧窗格中，选择应用。

  7. 如果要对本地文件夹进行上述更改，请选择文件 > 保存网络配置。

配置 Oracle Advanced Security 验证方法

配置 Oracle 高级安全性认证方法

集中的、安全的验证方法使您高度信任分布式环境中的用户、客户机和服务器的身份。网络验证方法还为用户提供了一次登入的好处。

使用 Oracle Net Manager 配置下列验证适配器：

  • Kerberos

  • CyberSafe

  • RADIUS

  • SSL

  • Windows NT Native 认证

注：如果配置了使用 SSL 的 TCP/IP 协议，在默认情况下将配置安全套接字层 (SSL)。在此选项卡中选择的任何验证方法都将覆盖 SSL 的验证功能。使用 SSL 选项卡配置其他的 SSL 选项。如果您正在使用带有 SSL 的 TCP/IP 协议，请不要使用任何其他验证方法。

要配置验证方法，请执行以下操作：

  1. 在导航器窗格中，展开 Oracle 网络配置 > 本地 > 概要文件。

  2. 从右侧窗格的列表中，选择 Oracle Advanced Security。

  3. 选择验证选项卡。

  4. 从可用方法列表中选择您要使用的验证方法，然后选择右箭头按钮 (>)。

  5. 根据您要使用的方法为验证方法排序。在所选方法列表中选择验证方法，并使用上移或下移按钮使所选方法在列表中上移或下移。

  6. 选择其他参数选项卡，为以下验证方法配置其他参数：

    • Kerberos

    • CyberSAFE

    • RADIUS

  8. 选择文件 > 保存网络配置。

为异构服务配置连接描述符

为异构服务配置连接描述符

异构服务使您能够从 Oracle 服务器访问非 Oracle 系统。要建立与非 Oracle 系统的连接，Oracle 服务器需要通过网关上的监听程序启动代理进程。为了使 Oracle 数据库连接到非 Oracle 系统，请使用网络服务名配置服务器。

要为异构服务配置服务器，请执行以下操作：

  1. 确保已为异构服务配置了监听程序，如为监听程序配置异构服务中所述。

  2. 在 tnsnames.ora 文件中创建可用于 Oracle 数据库服务器与非 Oracle 系统之间的连接的网络服务名。请确保连接描述符信息与为此监听程序配置的信息相匹配。

  3. 在本地 > 服务命名文件夹中选择网络服务名。

  4. 在右侧窗格的服务标识框中，选择高级。将显示“高级服务选项”对话框。

  5. 选择用于不同种类服务，然后选择确定。

  6. 在右侧窗格中，选择应用。

  7. 选择文件 > 保存网络配置。

为 Oracle Rdb 数据库配置连接描述符

为 Oracle Rdb 数据库配置连接描述符

Oracle Rdb 数据库是用于 Digital 的 64 位平台的数据库。Oracle Rdb 服务器对客户机的显示方式与 Oracle 数据库相同。Oracle Rdb 数据库有自己的监听程序。客户机配置包含使用 Oracle Rdb 特定参数创建一个网络服务名以访问数据库。

要配置 Oracle Rdb 数据库的客户机，请执行以下操作：

  1. 请创建网络服务名 (这在创建网络服务名中进行说明)。

  2. 在右侧窗格的服务标识框中，选择高级。将显示“高级服务选项”对话框。

  3. 在 Rdb 数据库字段中，输入 Oracle Rdb 数据库的文件名。

  4. 请随意在全局数据库名域中输入全局数据库名，并且如果需要的话，请在服务类型域输入服务类型。

  5. 在“高级服务选项”对话框中，选择确定。

  6. 在右侧窗格中，选择应用。

  7. 选择文件 > 保存网络配置。

有关详细信息，请参阅 Oracle Rdb 文档。

创建网络服务名

创建网络服务名

注：只有 OracleNetAdmins 或 OracleContextAdmins 组的成员用户可以在目录中创建网络服务名条目。

要向 tnsnames.ora 文件或目录服务器添加网络服务名，请执行以下操作：

  1. 在导航器窗格中，展开 Oracle 网络配置 > 目录/本地 > 服务命名。

  2. 从工具栏中选择加号 (+)，或从菜单栏中选择编辑 > 创建。将显示“网络服务名向导”的“欢迎使用”页。

  3. 在网络服务名字段中输入任意名称。网络服务名可使用客户机的域进行限定。如果指定了客户机域，网络服务名将自动限定域。

  4. 选择下一步。此时将显示“协议”页。

  5. 选择配置的监听程序进行监听的协议。请注意，此协议也必须安装在客户机上。

  6. 选择下一步。此时将显示“协议设置”页。

  7. 在提供的字段中为所选协议输入相应的协议参数。

  8. 选择下一步。此时将显示“服务”页。

  9. 选择一种发行版，输入目标服务，并且作为一种选择，可从 Oracle8i 或更高版本的“连接类型”列表中选择一种数据库连接类型。

    如果目标服务是 Oracle8i 或更高版本的服务，请选择 Oracle8i 或更高版本，并在服务名字段中输入服务名，然后选择下一步。如果目标服务是 Oracle8 或 Oracle7，请选择 Oracle8 或更低版本，在数据库 SID 字段中输入 Oracle 系统标识符，然后选择下一步。

    Oracle Corporation 建议对连接类型使用“数据库默认设置”。如果在初始化参数文件中配置了共享服务器，就可以选择专用服务器迫使监听程序绕过共享服务器配置来衍生一个专用服务器。如果在服务器上启用了“数据库驻留连接共享”，您可以选择池中服务器，从池中获取连接。如果在初始化参数文件中配置了共享服务器，并且希望确保此连接始终使用共享服务器，请选择共享服务器。

  10. 选择下一步。此时将显示“测试”页。
  11. 选择测试以验证网络服务名是否可用，或单击“完成”保存配置并关闭“网络服务名向导”。

    如果选择测试，Oracle Net 将通过使用您配置的连接描述符信息连接到数据库服务器。因此，要使测试成功，数据库和监听程序必须都在运行。如果没有运行，请参阅 Oracle10i Net Administrator's Guide 中的第 15 章“Establishing a Connection and Testing the Network”来启动这些组件。一个成功的测试将产生以下消息：

    连接测试成功。

    如果测试成功，请选择关闭以关闭“连接测试”对话框，然后继续步骤 12。

    如果测试不成功：

    a. 确保数据库和监听程序都在运行，然后选择测试。

    b. 选择“更改登录”以更改连接的用户名和口令，然后选择测试。

  12. 选择完成保存配置并关闭“网络服务名向导”。

  13. 请确保概要文件反映本地 (LOCAL) 或目录命名 (LDAP) 方法。
