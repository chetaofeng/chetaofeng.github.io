连接时故障转移

连接时故障转移

如果第一个监听程序未作出响应，客户机的连接请求将被转发给另一个监听程序。连接时故障转移由服务注册启用，因为监听程序在尝试连接前知道某个实例是否正在运行。

创建监听程序

创建监听程序

为了数据库接收连接，监听程序必须存在于服务器上。

注：在安装过程中，创建了名为 LISTENER 的默认监听程序。

要创建监听程序，请执行以下操作：

  1. a. 在导航器窗格中，展开 Oracle 网络配置 > 本地 > 监听程序。

  2. 在工具栏中选择加号 (+) 或选择编辑 > 创建。将显示选择监听程序名对话框。

  3. 接受监听程序名字段中提供的监听程序名，或用另一个名称编辑该字段。第一个监听程序的默认监听程序名是 LISTENER。后面的监听程序名默认为 LISTENERn。在任何给定的节点中监听程序名必须是唯一的。

  4. 选择确定。

  5. 完成配置监听程序地址中的步骤 4-6。

创建其他协议地址

创建其他协议地址

可以通过多个网络路由或协议地址访问数据库服务。

要将网络协议地址添加到现有网络服务名或数据库服务，请执行以下操作：

  1. 在导航器窗格中，展开 Oracle 网络配置 > 目录/本地 > 服务命名。

  2. 选择网络服务名或数据库服务。右侧窗格显示当前目标服务和地址列表。

  3. 在地址配置框中，选取加号 (+)。将显示新的地址选项卡。

  4. 请选择一个协议并为所选协议输入相应的协议参数信息。

  5. 根据地址列表中地址应处的位置用左箭头和右箭头按钮对地址进行排序。除非配置了多个地址选项，否则只联系列表中的第一个地址。要配置不同的次序，请参阅配置多个地址选项。

  6. 在右侧窗格中，选择应用。

  7. 如果要对本地文件夹进行上述更改，请选择文件 > 保存网络配置。

用服务名进行数据库标识

用服务名进行数据库标识

Oracle 数据库是作为服务呈现给客户机的，即数据库代表客户机执行工作。数据库可以有一个或多个与其有关的服务。

到 Oracle8i 为止，Oracle 数据库服务由 Oracle 系统标识符 (SID) 唯一标识。SID 也由数据库内部使用，以作为指向系统全局区 (SGA) 的指针。客户机通过指定连接描述符中的 SID 连接到数据库实例。此命名方案不区分实例的服务。

由于 Oracle 数据库可以分布到多个计算机上，因此在 Oracle9i 和 Oracle8i 中都指定了作为整体的服务和每个实例的服务。

服务名

数据库现在由其服务名标识。服务名由初始化参数文件中的 SERVICE_NAMES 参数指定。SERVICE_NAMES 指定可以跨实例和/或节点的最高级别的 Oracle 数据库服务视图的名称。SERVICE_NAMES 默认为全局数据库名，即由数据库名和域名组成的名称。

实例名

数据库实例由实例名以及初始化参数文件中的 INSTANCE_NAME 参数标识。INSTANCE_NAME 对应于实例的 SID。

连接描述符

用 SERVICE_NAME (没有 S) 参数配置连接描述符，以连接到 Oracle9i 或 Oracle8i 数据库。例如，以下连接描述符包含一个位于 sales1-sun 上的监听程序的地址，此监听程序监听名为 sales.us.example.com 的数据库服务的连接请求：

sales=

(DESCRIPTION=

(ADDRESS=(PROTOCOL=tcp)(HOST=sales-sun1)(PORT=1521))

(CONNECT_DATA=

(SERVICE_NAME=sales.us.example.com)))

也可以用 INSTANCE_NAME 参数配置连接描述符，以连接到数据库的特定实例。如果您具有带多个实例的 Oracle9i Real Application Clusters，可能需要使用这种方法。

例如，以下连接描述符包含一个位于 sales-sun1 上的监听程序的地址，此监听程序监听与 sales.us.example.com 数据库关联的名为 sales1 的实例的连接请求：

sales=

(DESCRIPTION=

(ADDRESS=(PROTOCOL=tcp)(HOST=sales-sun1)(PORT=1521))

(CONNECT_DATA=

(SERVICE_NAME=sales.us.example.com)

(INSTANCE_NAME=sales1)))

有关在连接描述符的 CONNECT_DATA 部分指定实例名的详细信息，请参阅 Oracle10i Net Services Administrator's Guide 中的第 13 章“Enabling Advanced Features for Oracle Net Services”。

修改数据库服务的网络路由信息

修改数据库服务的网络路由信息

注：只有 OracleNetAdmins 或 OracleContextAdmins 组的成员用户可以在目录中修改数据库服务的网络信息。

将数据库注册到目录中之后，Database Configuration Assistant 将在该目录中创建数据库服务条目。默认情况下，此条目包含网络路由信息，其中涉及到利用协议地址的监听程序的位置。如果已将此信息删除可重新创建，还可以修改现有的网络路由信息。

要创建或修改数据库服务的网络路由信息，请执行以下操作：

  1. 在导航器窗格中，展开 Oracle 网络配置 > 目录 > 服务命名。

  2. 选择数据库服务。右窗格显示当前目标服务名。

  3. 在地址配置框中，选取加号 (+)。将显示新的地址选项卡。

  4. 请选择一个协议并为所选协议输入相应的协议参数信息。

  5. 在右侧窗格中，选择应用。
