Oracle Context

Oracle Context

位于命名上下文或未发布的目录条目下的目录信息树 (DIT) 中的 cn=OracleContext 的相对唯一判别名。Oracle Context 包含用于 Oracle 软件的条目，例如用于目录命名的数据库服务和网络服务名。一个目录中可以有一个或多个 Oracle Context。

Oracle Internet Directory 自动在 DIT 结构的根目录中创建 Oracle Context。根 Oracle Context 的唯一判别名 (DN) 为(dn:cn=OracleContext)。

Oracle Net Manager 启动时，使用由 Oracle Net Configuration Assistant 配置的默认 Oracle Context。您可以将此 Oracle Context 更改为另一个环境。对于大多数部署来讲，需要创建附加的 Oracle Context。

Oracle 系统标识符 (SID)

Oracle 系统标识符 (SID)

一个名称，用于标识运行早于 Oracle 8i 的数据库的特定实例。对于任何数据库，都至少有一个引用数据库的实例。

对于早于 Oracle8i 的数据库，SID 被用来标识数据库。SID 作为连接描述符的一部分包括在 tnsnames.ora 文件中，并作为监听程序的定义包括在 listener.ora 文件中。应使用 SID 参数配置连接描述符，客户机将使用这些连接描述符连接到 Oracle8 或 Oracle7 数据库。

其他服务

其他服务

其他服务选项卡可用于指定监听程序将为其接收连接请求的非数据库服务。非数据库服务可以是外部过程或异构服务。

全局服务名称

输入此服务的服务名和域名。

程序名

输入可执行程序名。例如 extproc.exe。

程序参数零

输入内部第一个参数，它常作为代用程序名。例如：argv0

程序参数

输入应在启动服务时传送的命令行参数，例如，-mode=5。

环境

输入启动服务之前必须设置的任何环境变量。环境变量以下列格式指定：

environment_variable=values

例如：user=scott, start=immediate。

请注意，变量以逗号分隔。

SID

输入服务的程序名或 Oracle 系统标识符 (SID)。

Oracle 主目录

输入可执行程序的位置。

“添加服务”按钮

选择此按钮以配置非数据库服务信息。显示一个新的服务选项卡，可以在该选项卡中输入另一个服务的信息。

“删除服务”按钮

选择此按钮以删除选定的服务选项卡及其服务信息。

区分命名方法的优先次序

区分命名方法的优先次序

当配置好命名方法后，必须在概要文件中区分它们的优先次序。列表顶部的命名方法首选被使用，用于将网络服务名或数据库服务名解析为连接描述符。如果此方法不能解析名称，将使用列表中的第二个方法。

要指定命名方法的顺序，请执行以下操作：

  1. 在导航器窗格中，展开 Oracle 网络配置 > 本地 > 概要文件。

  2. 在右窗格的列表中，选择命名。

  3. 选择方法选项卡。

  4. 从可用方法列表中选择命名方法，并选择右箭头按钮。选定的命名方法就移动到选定方法列表中。

  5. 根据在解析网络服务名或数据库服务名时 Oracle Net 使用命名方法的先后顺序对其进行排序。从所选方法列表中选择命名方法，然后选择上移或下移上下移动列表中的选项。

  6. 选择文件 > 保存网络配置。

概要文件概览

概要文件概览

概要文件指定在客户机或服务器上启用和配置 Oracle Net 功能的首选项。使用概要文件，您可以：

  • 指定附加到未限定名后的客户机域

  • 区分命名方法的优先次序。

  • 启用事件记录和跟踪功能

  • 通过指定进程路由连接

  • 为外部命名配置参数

  • 配置 Oracle Advanced Security

要访问概要文件，在导航器窗格中，展开 Oracle 网络配置 > 本地 > 概要文件。

说明

默认值 = 1645
超时(秒)

Oracle 服务器等待主 RADIUS 服务器响应的时间 (秒)
默认值 = 15 秒
重试次数

Oracle 数据库服务器应向主 RADIUS 服务器发送消息的次数

缺省值为 3 次
机密文件 Oracle 服务器上的密钥文件位置。出于安全考虑，Oracle 建议您将此文件更改为仅限 root 用户访问。

建议的端口号

Oracle Corporation 建议将以下端口号用于 TCP/IP 和使用 SSL 的 TCP/IP 的协议地址：

端口

说明
1521

客户机连接到监听程序的默认监听端口在将来的发行版中，可能将此端口号更改为正式注册的端口号 2483 (TCP/IP) 和 2484 (使用 SSL 的 TCP/IP)。 

1575

 客户机连接到 Oracle Names Server 的默认和正式注册的监听端口。

1830

缺省的并已正式注册的监听端口，用于 Oracle Connection Manager 的管理命令。
