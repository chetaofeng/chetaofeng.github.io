修改数据库服务的网络路由信息

修改数据库服务的网络路由信息

注：只有 OracleNetAdmins 或 OracleContextAdmins 组的成员用户可以在目录中修改数据库服务的网络信息。

将数据库注册到目录中之后，Database Configuration Assistant 将在该目录中创建数据库服务条目。默认情况下，此条目包含网络路由信息，其中涉及到利用协议地址的监听程序的位置。如果已将此信息删除可重新创建，还可以修改现有的网络路由信息。

要创建或修改数据库服务的网络路由信息，请执行以下操作：

  1. 在导航器窗格中，展开 Oracle 网络配置 > 目录 > 服务命名。

  2. 选择数据库服务。右窗格显示当前目标服务名。

  3. 在地址配置框中，选取加号 (+)。将显示新的地址选项卡。

  4. 请选择一个协议并为所选协议输入相应的协议参数信息。

  5. 在右侧窗格中，选择应用。

数据库服务

数据库服务

数据库服务选项卡使您能够指定监听程序必须接收连接请求的数据库服务。监听程序需要该信息以将客户机连接到数据库。

Oracle9i 和 Oracle8i 实例使用动态服务注册来通知监听程序有关其数据库服务的信息。Oracle7 和 Oracle8 发行版 8.0 数据库以及 Oracle Enterprise Manager 工具要求 listener.ora 文件中的静态服务配置。

选择添加数据库以添加数据库服务信息。

全局数据库名

输入根据初始化参数文件中的 SERVICE_NAMES 参数值或 DB_NAME 和 DB_DOMAIN 参数值提供的数据库名和域名。

Oracle 主目录

输入数据库的 Oracle 主目录位置。

注：Windows NT 已经知道 Oracle 主目录。因此，连接 Windows NT 上的服务不需要此字段。

SID

输入实例的 Oracle 系统标识符 (SID)。

“添加数据库”按钮

选择此按钮为另一个数据库配置服务信息。此时将显示新的数据库选项卡，可以在该选项卡中输入另一个数据库的服务信息。

删除数据库按钮

选择此按钮以删除所选的数据库选项卡及其服务信息。

用服务名进行数据库标识

用服务名进行数据库标识

Oracle 数据库是作为服务呈现给客户机的，即数据库代表客户机执行工作。数据库可以有一个或多个与其有关的服务。

到 Oracle8i 为止，Oracle 数据库服务由 Oracle 系统标识符 (SID) 唯一标识。SID 也由数据库内部使用，以作为指向系统全局区 (SGA) 的指针。客户机通过指定连接描述符中的 SID 连接到数据库实例。此命名方案不区分实例的服务。

由于 Oracle 数据库可以分布到多个计算机上，因此在 Oracle9i 和 Oracle8i 中都指定了作为整体的服务和每个实例的服务。

服务名

数据库现在由其服务名标识。服务名由初始化参数文件中的 SERVICE_NAMES 参数指定。SERVICE_NAMES 指定可以跨实例和/或节点的最高级别的 Oracle 数据库服务视图的名称。SERVICE_NAMES 默认为全局数据库名，即由数据库名和域名组成的名称。

实例名

数据库实例由实例名以及初始化参数文件中的 INSTANCE_NAME 参数标识。INSTANCE_NAME 对应于实例的 SID。

连接描述符

用 SERVICE_NAME (没有 S) 参数配置连接描述符，以连接到 Oracle9i 或 Oracle8i 数据库。例如，以下连接描述符包含一个位于 sales1-sun 上的监听程序的地址，此监听程序监听名为 sales.us.example.com 的数据库服务的连接请求：

sales=

(DESCRIPTION=

(ADDRESS=(PROTOCOL=tcp)(HOST=sales-sun1)(PORT=1521))

(CONNECT_DATA=

(SERVICE_NAME=sales.us.example.com)))

也可以用 INSTANCE_NAME 参数配置连接描述符，以连接到数据库的特定实例。如果您具有带多个实例的 Oracle9i Real Application Clusters，可能需要使用这种方法。

例如，以下连接描述符包含一个位于 sales-sun1 上的监听程序的地址，此监听程序监听与 sales.us.example.com 数据库关联的名为 sales1 的实例的连接请求：

sales=

(DESCRIPTION=

(ADDRESS=(PROTOCOL=tcp)(HOST=sales-sun1)(PORT=1521))

(CONNECT_DATA=

(SERVICE_NAME=sales.us.example.com)

(INSTANCE_NAME=sales1)))

有关在连接描述符的 CONNECT_DATA 部分指定实例名的详细信息，请参阅 Oracle10i Net Services Administrator's Guide 中的第 13 章“Enabling Advanced Features for Oracle Net Services”。

数据库链接

数据库链接

定义从 Oracle 数据库服务器到另一个数据库服务器单向通信路径的指针。链接指针实际上被定义为数据字典表中的一个条目。要访问该链接，必须连接到包含数据字典条目的本地数据库。

“数据库链接”连接是单向的：它允许连接到本地数据库 A 的客户机使用存储在数据库 A 中的链接访问远程数据库 B 中的信息，但是，连接到数据库 B 的用户不能使用同一链接访问数据库 A 中的数据。如果数据库 B 上的本地用户要访问数据库 A 上的数据，他们必须定义存储在数据库 B 的数据字典中的链接。

支持下列数据库链接类型：

  • 在数据库特定方案中的私有数据库链接。只有私有数据库链接的所有者可以使用它。

  • 数据库的公共数据库链接。数据库的所有用户都可以使用它。

有关创建私有和公共数据库链接的详细信息，请参阅 Oracle10i Database Administrator's Guide。

目录命名

目录命名

一种命名方法，它将数据库服务或网络服务名解析为存储于中心目录服务器的连接描述符。

目录提供对数据库服务和网络服务名的集中管理，减少有关添加和重定位服务的工作。虽然网络服务名可以配置成一个服务的别名，但目录也可以不使用网络服务名而直接引用数据库服务。安装期间，数据库服务作为一个条目被自动添加到目录中。

目录命名配置概览

安装期间的服务器配置

在进行“定制”服务器安装期间，Oracle Net Configuration Assistant 将提示您：

  • 选择目录服务器的类型

  • 标识目录的位置

  • 选择或创建服务器能够在其中查找、创建和修改连接标识符的 Oracle Context

  • 配置目录命名作为优先使用的命名方法

如果正确配置了目录用法，Database Configuration Assistant 将尝试连接到该目录并在目录中注册数据库服务条目。数据库服务条目是使用连接描述符配置的，其中包含以下网络信息：

  • 监听程序地址

  • 数据库服务的名称

安装期间的客户机配置

在所有客户机安装期间，Oracle Net Configuration Assistant 都将提示您配置以下目录用法设置：

  • 指定目录类型

  • 标识目录的位置

  • 选择客户机能够在其中查找连接标识符的默认 Oracle Context

  • 配置目录命名作为优先使用的命名方法

Oracle Net Configuration Assistant 在安装后进行的配置

Oracle Net Configuration Assistant 还可以在独立模式下运行，以执行以下操作：

  • 配置客户机或服务器 (如上文所述)

  • 创建 Oracle Schema

  • 创建 Oracle Context

要以独立模式运行 Oracle Net Configuration Assistant，请执行以下操作：

  • 在 UNIX 上，运行 $ORACLE_HOME/bin 下的 netca。

  • 在 Windows NT 上，选择开始 > 程序 > Oracle - HOME_NAME > 网络管理 > Oracle Net Configuration Assistant。

具有 Oracle Net Manager 的配置

Oracle Net Manager 使您能够完成以下操作：

  • 更改 Oracle Context

  • 更改验证身份证明

  • 创建网络服务名

  • 创建网络服务别名

  • 通过多个协议地址配置到监听程序的多个路由

  • 配置数据库服务或网络服务名的高级 CONNECT_DATA 信息。

  • 将网络服务名条目从 tnsnames.ora 文件移植到目录服务器。

要访问目录命名配置，请在导航器窗格中选择目录服务命名。

有关详细信息，请参阅 Oracle Net Configuration Assistant 帮助系统。

安装期间的服务器配置

全部驻留在一个目录服务器上的子树。它是相邻的子树，也就是，它必须以作为子树顶级的条目开始，并向下扩展出叶条目或对从属命名上下文的引用条目。它的大小可以从一个条目到整个目录信息树 (DIT)。

Oracle Context 可以在一个命名上下文之下创建。

专用服务器

一种需要每个用户进程对应一个专用服务器进程的服务器。每个客户机拥有一个服务器进程。Oracle Net 把现有服务器进程地址发回客户机。然后，客户机将其连接请求重新发送到提供的服务器地址。与共享服务器相对。

目录服务器

“目录服务器验证”对话框

“目录服务器验证”对话框

“目录服务器验证”对话框使您能够为在目录服务器中选定的 Oracle Context 设置验证身份证明。

必须使用目录服务器对 Oracle Net Manager 进行验证，以便执行任何目录服务器操作，如查看、添加或修改目录中的 Oracle Net 条目。

用户：

输入具有目录访问权的授权用户。

若要添加或修改条目，请以唯一判别名 (DN) 形式输入已被授予添加和修改权限的目录管理员用户。例如，c=US,o=EXAMPLE,ou=admin,cn=scott 是名为 Scott 的管理员的 DN。

口令

输入此用户的口令。

注：如果想用“匿名”权限设置简单视图对象和目录，则将“ 用户和口令”字段留空。

在默认情况下，Oracle Net Manager 试图用以下方法之一进行目录验证：

  • Native 认证：使用 Windows NT 操作系统的用户名和口令。

  • 强力验证：使用安全套接字层 (SSL) 认证。
