共享服务器

共享服务器

一种配置的服务器，允许有更多的用户进程共享很少的服务器进程的，这样可以增加所支持的用户数。有了共享服务器配置，许多用户进程可以连接到同一个调度程序。调度程序将多个入网网络会话请求指引到一个公用队列。服务器进程共享池中某个闲置的共享服务器进程从队列中获得一个请求。这意味着一个小服务器进程池就可以对大量客户机提供服务。与专用服务器相反。

sqlnet.ora 文件

sqlnet.ora 文件

一种用于客户机或服务器的配置文件，该文件指定：

  • 附加到未限定的服务名或网络服务名的客户机域

  • 命名方法的次序，在解析名称时客户机需要使用到这些方法。

  • 要使用的事件记录和跟踪功能

  • 连接路径

  • 首选的 Oracle Names Server

  • 外部命名参数

  • Oracle Advanced Security 参数

sqlnet.ora 文件必须驻留在以下某一位置：

  1. 由环境变量 TNS_ADMIN 指定的目录。如果在 Windows NT 上没有将 TNS_ADMIN 环境变量定义为变量，它可能在注册表中。

  2. 在 UNIX 上为 $ORACLE_HOME/network/admin，而在 Windows 操作系统上为 ORACLE_HOME\network\admin 目录。

静态配置数据库服务信息

静态配置数据库服务信息

为使监听程序能把客户机请求接收到 Oracle8 或 Oracle7 数据库，必须把数据库的信息配置到 listener.ora 文件中去。

如果数据库是 Oracle9i 或 Oracle8i 数据库，则 listener.ora 文件中不需要静态服务信息。服务信息通过服务注册自动注册到监听程序。不过，如果正在使用 Oracle Enterprise Manager，则需要 Oracle8i 数据库的静态信息。

要配置监听程序的数据库服务信息，请执行以下操作：

  1. a. 在导航器窗格中，展开 Oracle 网络配置 > 本地 > 监听程序。

  2. 选择监听程序。

  3. 从右侧窗格的列表中，选择数据库服务。

  4. 选择添加数据库。出现一个新的数据库选项卡。

  5. 在全局数据库名字段中输入全局数据库名，在 Oracle 主目录字段中输入 Oracle 主目录的位置，在 SID 字段中输入数据库的 Oracle 系统标识符 (SID)。

  6. 选择文件 > 保存网络配置。

测试连接性

测试连接

通过执行连接测试，您可以验证一个选定的服务命名对象到数据库服务的连接。在连接测试期间，使用已配置的连接描述符信息实现对数据库服务的连接。

要验证网络连接信息，请执行以下操作：

  1. 确保监听程序和数据库正在运行。

  2. 在导航器窗格中，展开 Oracle 网络配置 > 目录/本地 > 服务命名。

  3. 选择一个现有的网络服务名或数据库服务。

  4. 从任务栏选择数据库符号，或选择命令 > 测试服务。此时将显示“连接测试”对话框，当有测试结果产生时会进行更新。一个成功的测试将产生以下消息：

连接测试成功。

有关启动组件的信息，请参阅 Oracle10i Net Administrator's Guide 中的第 15 章“Establishing a Connection and Testing the Network”。

tnsnames.ora 文件

跟踪

跟踪

一种将有关某个操作的详细信息写入到输出文件的功能。跟踪功能生成语句的详细序列，这些语句说明操作执行时的操作事件。管理员使用跟踪功能诊断异常情况；通常不启用此功能。与通过事件记录提供的信息相比，跟踪操作可使您获得更多的有关组件内部操作的信息。

注：跟踪使用大量磁盘空间，可能对性能产生重大影响。因此，只在必要时才启用跟踪。

“创建网络服务别名”对话框

“创建网络服务别名”对话框

使用“创建网络服务别名”对话框可以在目录服务器中创建 网络服务别名。

网络服务别名是网络服务名或数据库服务的替代名称。网络服务别名条目没有连接描述符信息。它只引用使用别名所表示的对象的位置。当客户机请求对网络服务别名进行目录查找时，此目录确定此条目为网络服务别名，然后象查找所引用的条目那样完成查找。例如，可以为数据库服务 db1 创建网络服务别名 db1alias。当 db1alias 用于连接数据库服务时，如 CONNECT scott/tiger@db1alias，它实际上将被解析并使用 db1 的连接描述符信息。

网络服务别名有两种主要用途：

  • 使用网络服务别名，以便客户机可按另一名称引用数据库服务或网络服务名。

  • 将一个 Oracle Context 中的网络服务别名用于另一 Oracle Context 中的数据库服务或网络服务名。这样，数据库服务或网络服务名只需在目录服务器中定义一次即可被使用其他 Oracle Context 的客户机引用。

网络服务别名

输入网络服务别名的名称。

网络服务名或数据库服务

使用网络服务名或数据库服务框可以输入有关数据库服务或网络服务名的信息：

Oracle Context：从列表中选择一个值或在字段中输入一个值，以此指定数据库服务或网络服务名的 Oracle Context。

名称：从列表中选择一个值或在字段中输入一个值，以此指定数据库服务或网络服务名的唯一判别名 (DN)。
