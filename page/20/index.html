路由连接请求

路由连接请求

可以配置客户机以使连接请求总是由专用服务器进程或者 Oracle Connection Manager 负责。

要路由连接请求，请执行以下操作：

  1. 在导航器窗格中，展开 Oracle 网络配置 > 本地 > 概要文件。

  2. 从右侧窗格的列表中选择一般信息。

  3. 选择路由选择选项卡。

  4. 选择希望路由连接请求的首选方式。

  5. 选择文件 > 保存网络配置。

服务标识

服务标识

使用此属性工作表可以查看或修改所选网络服务名或数据库服务的网络信息。

服务标识

使用“服务标识”框查看或修改 CONNECT_DATA 信息。

元素

说明
服务名

如果将此客户机配置成连接 Oracle8i 或更高版本的数据库服务，则输入服务名。
注：对于存储在目录服务器上的数据库服务，该字段仅显示当前值。不允许进行修改。

SID

 显示 Oracle8 数据库的 Oracle 系统标识符 (SID)。
连接类型

在列表中，选择是希望此网络服务名使用专用服务器、共享服务器，还是为数据库配置的任何服务器。
如果在初始化参数文件中配置了共享服务器，则选择“专用服务器”可以强制监听程序启动专用服务器以绕过共享服务器配置。
如果在初始化参数文件中配置了共享服务器，并且希望确保此连接始终使用共享服务器，则选择“共享服务器”。如果在服务器上启用了“数据库驻留连接共享”，则可以选择池中服务器，以从池中获取连接。

使用 Oracle8 发行版 8.0 兼容标识 如果正在配置连接到早于 Oracle8i 发行版的数据库服务的客户机，请选择此选项。如果正在配置 Oracle9i 或 Oracle8i 数据库服务，请清除此选项。
高级 选择此项以配置高级的 CONNECT_DATA 参数。

地址配置

使用“地址”框修改、添加或删除监听程序的协议地址。

如果地址已配置好，则它的协议信息将在字段中显示。

元素

说明
协议

从列表中，选择监听程序地址的协议。依据所选择的协议输入相应的信息。
“显示高级/隐藏高级”按钮

使用“显示高级/隐藏高级”按钮可进行切换，以便显示或隐藏为地址指定的 I/O 缓冲区空间限制的值。

TCP/IP、启用 SSL 的 TCP/IP 以及 SDP 协议都支持这些设置。

  • 发送缓冲区大小总计：指定会话发送操作的缓冲区空间，单位是字节。
  • 接收缓冲区大小总计：指定会话接收操作的缓冲区空间，单位是字节。

注：指定这些设置后将覆盖客户机端 sqlnet.ora 文件中的相应设置。

+ 按钮

 选择此项以创建新的协议地址。此时将显示一个新的地址选项卡，可以在该选项卡中选择协议，并在字段中输入协议信息。客户机和服务器 (驻留有监听程序) 上都必须已经安装所选择的协议。
X 按钮

选择此选项以删除所选的地址选项卡及其地址信息。
< 按钮

选择此项以在列表次序中将协议地址上移。

> 按钮		 选择此项以在列表次序中将协议地址下移。
“高级”按钮 选择此项以配置多个地址选项，诸如连接时故障转移和客户机负载平衡。

“应用”按钮

选择此项以应用这些设置。

“还原”按钮

选择此项以将设置重置为原始值。

有关配置 I/O 缓冲区空间的详细信息，请参阅 Oracle10i Net Administrator's Guide 中的第 14 章“Optimizing Performance”。

任务概览

任务概览

以下是说明如何为网络配置 Oracle 高级安全性的一系列步骤：

  1. 选择认证方法。

  2. 区分在客户机/服务器协商过程中将使用的认证方法的优先次序。

  3. 为选定的每种认证方法配置其他参数。

  4. 配置客户机端和服务器端的完整性 (校验和级)。

  5. 配置客户机端和服务器端加密。

  6. 配置客户机端和服务器端的安全套接字层 (SSL) 通讯设置。
