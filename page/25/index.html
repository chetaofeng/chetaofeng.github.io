搜索

查找 Oracle Names Server 的过程。在搜索过程中，定位一个 Oracle Names Server。在找到第一个 Oracle Names Server 之后，该 Oracle Names Server 试通管理区域内的所有其他 Oracle Names Server。接着创建 Oracle Names Server 的列表并把它保存到.sdns.ora (UNIX 上) 或 sdns.ora (Windows 上)。按响应时间次序把该列表排序。

“搜索”按以下次序搜索第一个 Oracle Names Server ：

  1. sqlnet.ora 文件中首选的 Oracle Names Server

  2. 众所周知的 Oracle Names Server

  3. 本地 Oracle Names 服务器在端口1575 配置 TCP/IP。

有关搜索 Oracle Names Server 的信息，请参阅“Discover other Oracle Names Servers”。

目录命名概览

目录命名概览

目录命名是主要的命名方法之一，用来将连接标识符解析为客户机用来连接到数据库的连接描述符。目录命名将连接标识符及其连接描述符存储在符合 LDAP 的目录服务器中。连接标识符存储在包含用于 Oracle 软件的条目的 Oracle Context 子树下。

最终用户按下列格式输入包括连接标识符的连接字符串：

CONNECT username/password@connect_identifier

在下例中，sales 是在目录中进行查找的连接标识符：

CONNECT scott/tiger@sales

返回到客户机的连接描述符包括：

  • 监听程序协议地址的位置

  • 数据库的服务名

客户机使用协议地址信息来连接到监听程序。用同一个地址信息配置的监听程序接收此请求。如果监听程序正在监听对同一数据库服务的请求 (该数据库服务在连接描述符中指定)，将授予一个连接。

对于要进行的目录命名，请配置 Oracle Net Services 功能的目录服务器，并配置访问此目录的客户机。

有关详细信息，请参阅 Oracle10i Net Administrator's Guide 中的第 3 章“Configuration Management Concepts”。

将 Oracle Names 条目导出到目录服务器

此发行版中不再支持 Oracle Names。如果要使用 9.2 版的 Oracle Names，必须将存储在 Oracle Names Server 中的数据库和别名对象导出到目录服务器，或间接导出到 LDAP 数据交换格式 (LDIF) 文件中，随后再利用该文件来加载目录服务器。在导出数据库对象时，仅导出地址记录 (A.SMD)。地址记录中包含数据库对象的连接描述符信息。数据库对象是以网络服务名的方式导出的。别名是以网络服务别名的方式导出的。数据是从指定的域导出的。

有关详细信息，请参阅 Oracle10i Net Administrator's Guide 中的第 8 章“Configuring Naming Methods”。

有关自 Oracle Names Server 导出数据的完整说明，请参阅发行版 9.2 中的 Oracle9i Net Services Administrator's Guide。

将 tnsnames.ora 文件条目导出到目录服务器

将 tnsnames.ora 文件条目导出到目录服务器

已在 tnsnames.ora 文件中配置的网络服务名可导出到目录服务器中。一次只能对一个域执行导出。

导出 tnsnames.ora 文件时，将在该目录中当前选定的 Oracle Context 中创建网络服务名。如果需要更改 Oracle Context，请选择命令 > 目录服务器 > 更改 Oracle Context。

要将存储在 tnsnames.ora 文件中的网络服务名导出到一个目录服务器，请执行以下操作：

  1. 选择命令 > 目录 > 导出服务名。此时将启动“目录移植向导”。

  2. 选择下一步。

    如果检测到多个域，则出现“选择域”页。接着进行步骤 3。

    如果网络服务名不是域限定的，将显示“选择网络服务名”页。跳到步骤 4。

  3. 选择您希望导出其网络服务名的网络域，然后选择下一步。此时将显示“选择网络服务名”页。

  4. 从列表中选择要导出的网络服务名，然后选择下一步。此时将显示“选择目标上下文”页。

    在“选择目标上下文”页中执行下列步骤：

    a. 从目录命名上下文列表中选择包含 Oracle Context 的目录命名上下文。目录命名上下文是包含一个或多个 Oracle Context (它包含 Oracle 条目) 的目录子树的一部分。

    b. 从 Oracle Context 列表中选择您要将选定的网络服务名导出到其中的 Oracle Context。

    c. 选择下一步。此时将显示带有导出操作状态的“目录服务器更新”页。

  5. 选择完成关闭“目录服务器移植向导”。
